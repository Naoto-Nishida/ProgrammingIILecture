    void DrawCircle(int xi, int yi, int radius)
    {
        int i;
        const int nDivision = 16;
     
        glBegin(GL_TRIANGLE_FAN);
        glVertex2i(xi, yi);
        for (i=0; i<=nDivision; i++)
        {
            const double radian = 2*M_PI*(i/(double)nDivision);
            const double x = cos(radian) * radius + xi;
            const double y = sin(radian) * radius + yi;
            glVertex2d(x, y);
        }
        glEnd();
    }

    • CurveErasePoint 関数を実装するときは、1. リストの要素が何もない場合、2. クリックされたのがリストの先頭の頂点の場合、3. クリックされたのがリストの末尾の頂点の場合、4. クリックされたのがそれ以外の場合、の 4 通りに場合分けするとよい。
    • ファイルに折れ線のデータを読み書きする場合、各折れ線の頂点数を記録したくなるかもしれない。それもよいが、リスト構造の要素数を調べるにはいちいちリスト構造の先頭から末尾まで辿る必要がある。頂点数を記録する代わりに、各頂点の座標を記録した最後に、座標として使われない値 -1 を記録すればよい。そして読み込みの際には、-1 が現れたら頂点の座標の読み込みが終わった、とすればよい。このように終端を表す特別な値を番兵という。文字列の末尾のヌル文字 '\0' やリスト構造の末尾の NULL なども番兵の一種である。

レポートに関して類似した質問が多くあった場合、以下にその質問と回答を掲載します。定期的にチェックしてください。
Q. 自分の PC や遠隔ログインで OpenGL を使いたいのですがどうすればいいですか？
A. 第 6 回のレポート課題の Q & A を見てください。
Q. GLUT に必要なヘッダファイルやライブラリファイルはコピーしたのに、ソースコードをコンパイルしようとすると、次のようなエラーが出ます。
$ cc -c circles.c
circles.c:4:21: 致命的エラー: GL/glut.h: そのようなファイルやディレクトリはありません
A. コンパイラがヘッダファイルを探しに行けていないのが問題だと思われます。-I. (ハイフン、大文字の i 、ピリオドをくっつけたもの) をオプションに加えてコンパイルしてみてください。-I はコンパイラがヘッダファイルを探索する場所を指定するためのオプションです。ピリオドはカレントディレクトリを表します。 
$ cc -c circles.c -I.
Q. 課題 8-2 で、コンパイルは通ったのに実行したらいきなり Segmentation Fault が出てクラッシュします。
A. CurveDraw 関数に問題がないか確認してください。みなさんが実装する関数のうち、プログラムが起動して最初に呼ばれるのが CurveDraw 関数です。例えば、プログラム起動時は curve は空のはずなのにそれを無視して描画しようとするとクラッシュします。その場合は curve が空 (CurveIsEmpty 関数の返り値が真) なら何もせずに return する、という処理を最初に入れてください。

授業の進め方

  • 授業 Web ページに各回の実習内容を「実習テキスト」として載せる。各人、実習テキストの指示に従って実習を進めること。
  • 実習テキストのチェック問題は成績評価の対象とする。原則として実習テキストの基本問題はすべて各回の実習時間中に終えるものとし、チェック問題ごとに TA または教員にチェックを依頼すること。
  • 各回にレポート課題を課す。受講者はレポート課題を期限までにレポートにまとめて提出すること。レポートの提出方法は下記「レポートの提出方法」を参照のこと。
  • 本実習に用いる計算機は Linux である。受講者は、授業開始前に、自分に割り当てられた計算機の Linux を起動しておくこと。

成績評価およびレポート作成・提出方法など

成績評価

この実習では次の 3 つに基づいて成績を評価する。
  • 各回の基本問題の進捗状況
  • レポート（各回 100 点満点、チャレンジ問題を解けば別途加点）
レポート採点方針
  • 各課題の要件を満たしているか、学習した内容が理解できているかを評価する。
  • 同一内容とみなされるものが見つかった場合は、manaba 上で最初に提出されたもののみ採点し、他はすべて 0 点とする。
なお、やむを得ない理由により遅刻・欠席した場合は、「欠席届」を提出し理由を説明すれば配慮する。

レポート作成について

レポートの作成方法は、基本的には「プログラミング実習 IA・IB」のレポートの作成方法と同じである。
レポートの報告内容
  • 作成したソースコード
    全学計算機上でのソースコードのフルパスは記載しなくてよい。
  • コンパイルのためのコマンド
    ソースコード名は main 関数があるファイルについて a1-1.c のように頭文字 a につづき課題番号名.c とする。その他のファイルについては特に規定しない。授業の後半には複数ファイルに分割してコンパイルすることもあるので、どのようにコンパイルしたか分かるようにし、採点者がファイルをコピーした後、コンパイルできるようにすること。
  • 実行結果
    要件を満たしていることを示すのに十分な実行例を示すこと。課題に示されている実行例も正しい結果が得られたことを示すものとして必要である。しかし、それ以外にも、境界値（例：添え字の下限、読み込むべきデータ量の上限、場合分けが起こる値）を用いた実行例も、仕様が指定している量のデータを正しく処理できることを示す、正しく場合分けが扱えていることを示す等のために必要である。そのような境界値が存在しない場合にも、正しい実行例がたまたま得られているのではないことを示すために、実行例に示されている以外に数例を示す必要があるだろう。
  • その他（レポートを読むだけで、正しく解答できていることを確認できる情報）
課題で作成するプログラムについての注意事項
  • 原則として、作成したプログラムは実習室の計算機の Linux 上で動作するようにすること。
  • チャレンジ問題などで実習室の計算機にインストールされていないライブラリなどを用いた場合は、プログラムのコンパイルおよび実行に必要なライブラリの詳細 (名前、バージョン、配布元 URL など) と、実装・実行環境 (コンパイラの種類、バージョン、OS など) をレポートに明記すること。
  • グローバル変数は使ってよいと明記されていない限り、使わないこと。

レポート提出方法

レポートは紙を提出するのではなく、manaba 上のコースに下の提出物をすべてアップロードすること。なお提出物に不足がある場合、採点しない、もしくは減点することがある。
提出物
  • レポートの PDF ファイル
    ファイル名を「自分の学籍番号＋ハイフン＋自分の氏名（アルファベット）＋ハイフン＋課題番号2桁＋".pdf"」とすること。
    例：201400112-HanakoTsukuba-03.pdf
    ファイル名に用いることが可能な文字は半角文字のみである。また氏名に空白文字を使わないこと。
  • 作成したプログラムのソースコードなど、コンパイルおよび実行に必要なファイル一式を含む zip ファイル
    zip ファイルのファイル名は、上記の PDF のファイル名と同様に「自分の学籍番号＋ハイフン＋自分の氏名（アルファベット）＋ハイフン＋課題番号2桁＋".zip"」とすること。
    例：201400112-HanakoTsukuba-03.zip
    「プログラミング実習 IA・IB」と異なり、「プログラミング実習 II」ではファイル数が増えがちなので、zip ファイルで圧縮して提出すること。ただし実行ファイルは提出しなくてよい。もし zip ファイルのファイルサイズが数 MB 以上になった場合 (最終課題などではファイルサイズが増えることが予想される)、Dropbox や OneDrive などにアップロードし、その URL をレポートに記載すること。
提出期限を、次回授業の前日の 15:15とする。これ以外になる場合には別途指示する。なお、提出期限までは何回でも提出し直すことが可能である。したがって、提出期限の十分前にある程度仕上がった状態で一旦提出し、その後、完成度を高めていきながら再提出を繰り返す戦略を薦める。なぜなら、この戦略を採れば提出を確実に行うことおよび完成度を高めることを両立させられるからである。

質問について

どうしても分からないところがあれば、詳しい友人、TA、教員に積極的に質問・相談することを強く薦める。情報系では今後の他の科目でも、自力では行き詰まってしまうことがあるが、詳しい人に聞けばすぐ解決することも多い。この実習を通じて質問・相談する習慣を身につけてほしい。ただし、友人と相談しても全く同一のレポートにならないように注意すること。
メールその他の方法で TA や教員に質問する際は、次の項目に従ってほしい。
  • 自分の名前を書く。
  • 単に「うまく動かない」ではなく、どういう症状なのかを説明する。実行結果も示す。
  • 再現できるように、ソースコード一式と必要なデータを添付する。
  • うまくいってもいかなくても、結果がどうなったか、あとで報告する。質問を受けた方も、結果がどうなったか気になっている。
教員や TA のメールアドレスについては、このページの「担当教員/担当 TA」を参照すること。

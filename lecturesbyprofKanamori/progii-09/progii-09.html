キーボード入力の受け付け、マウス入力の受け付け

キーボードからの入力を受け付けるには、次のような引数を持つコールバック関数を登録すればよい。登録には glutKeyboardFunc の引数としてコールバック関数を指定する。 
void keyboard(unsigned char key, int x, int y);
この keyboard 関数の引数について、key は押されたキー、x と y は画面の左下を原点としたときのマウスカーソルの座標である。 この他に、ファンクションキーやカーソルキー (↑↓←→) などの特殊キーを扱いたい場合は、また別のコールバック関数を用意する。登録には glutSpecialFunc の引数としてコールバック関数を指定する。 
void special(int key, int x, int y);
  • 参考: GLUT で扱える特殊キー一覧
マウス処理については、マウスクリック時、ドラッグ時、クリックもドラッグもせずマウスを動かしている時、の 3 つについて、それぞれコールバック関数を登録できる。それぞれ、glutMouseFunc, glutMotionFunc, glutPassiveMotionFunc でコールバック関数を登録する。 
/* マウスクリック時に呼び出されるコールバック関数 */
void mouse(int button, int state, int x, int _y);

/* マウスドラッグ時に呼び出されるコールバック関数 */
void motion(int x, int _y);

/* クリックもドラッグもせずマウスを動かしている時に呼び出されるコールバック関数 */
void passive(int x, int _y);
上記の関数で、x, _y はそれぞれマウスの座標である。ただし y 座標については注意が必要で、画面の左上が原点として座標が与えられるので、y 座標のみ上下反転して処理する必要がある。

プログラム内で時間を測定する方法、画像の表示方法

プログラム内で時間を測定する方法

経過時間に応じて少しずつ画面の表示を変え、アニメーションさせるには、プログラム内で時間を測定する必要がある。 なめらかなアニメーションを行うには、1/60 秒ごと、つまりおよそ 17 ミリ秒ごとに画面を更新せねばならない。 プログラムの実行時間を測定するには、以前は time コマンドを利用したが、これではプログラムが実行中の時間を取得することはできない。また、乱数を生成するときに time 関数を使ったが、time 関数は秒単位の精度しかない。
では、ミリ秒単位で時間を測るにはどうしたらよいかというと、Unix 系の OS (Linux や MacOS) では、gettimeofday 関数が利用できる。Windows にはこの関数がなく、代わりに QueryPerformanceCounter という関数を用いる。これらの関数は使い方がやや煩雑なため、この実習ではそれらを使いやすくした関数を用意した。
  • 時間の計測を開始する関数 StartTimer 
    void StartTimer(void);
  • 計測開始から現在までの経過時間をミリ秒単位で返す関数 GetTime 
    double GetTime(void);
  • ある基準時刻 prevTime (計測開始からのミリ秒単位の時刻) からの経過時間を返す関数 GetRapTime 
    double GetRapTime(double prevTime);
これらは GenericTimer.c/.h ファイルに記述されている (後述のサンプルコードに同梱)。 使い方は、例えば、まず初期化処理を行う init 関数の中で、StartTimer を呼んでおく。 
#include "GenericTimer.h"

void init(void)
{
    /* 何か初期化処理を記述する */

    StartTimer();
}
そして、ある処理にかかった時間を計測したければ、例えば次のように書くと prevTime からの経過時間が rapTime にミリ秒単位で記録される。 
double prevTime, rapTime;

prevTime = GetTime();

/* ここで何かの処理を行う */

rapTime = GetRapTime(prevTime);

画像の表示方法

これまでに PPM 形式の画像の読み書きを学んだので、ファイルから読み込んだ画像をウィンドウに表示できるようにしてみよう。 画像の読み込みには、第 7 回で作った画像ライブラリに少し手を加えたものを用意した (後述のサンプルコードに同梱)。
  • 画像データの初期化を行う関数 InitImageData 
    void InitImageData(ImageData *img);
    この関数はメンバ変数 data に NULL を代入する。img->data が NULL かどうかで、画像データのメモリ確保が行われたかどうかを判定できるようになる。

  • 画像データのメモリ確保が行われたかどうかを判定する関数 IsImageDataAllocated 
    int IsImageDataAllocated(ImageData *img);
    img->data に NULL が入っているかどうかで、メモリ確保が行われたかどうかを判定する。

  • 画像データのメモリ確保とメンバ変数の代入を行う関数 AllocateImageData 
    int AllocateImageData(ImageData *img, int _w, int _h, int _channels);
  • 画像データのメモリの解放を行う関数 ReleaseImageData 
    void ReleaseImageData(ImageData *img);
  • 画像データをコピーする関数 CopyImageData 
    void CopyImageData(ImageData *imgIn, ImageData *imgOut);
  • 画像データを上下反転する関数 FlipImageData 
    void FlipImageData(ImageData *img);
    PPM 画像では左上が原点だが、GLUT では左下が原点になるので、画像を上下反転させる必要がある。

  • 画像データを描画する関数 DrawImageData 
    void DrawImageData(ImageData *img, int xi, int yi);
    画像の左下が座標 (xi, yi) に一致するように移動して、画像データをウィンドウに表示する。

上記のうち、画像データをウィンドウに表示する関数 DrawImageData について補足する。OpenGL で画像データを表示するには、(1) ピクセルの値を直接表示する (glDrawPixels 関数を用いる) のと、(2) 画像データをテクスチャというデータ形式に変換して物体に貼りつけて表示する、という 2 通りがある。後者の方がより汎用的だが、扱いが煩雑になるため、DrawImageData 関数の中では前者を使っている。

テクスチャを用いた画像の表示方法

上に書いた DrawImageData 関数を使った場合、画像の四隅が画面からはみ出すと画像が消えてしまう。以下のテクスチャを使った画像の表示ならそのような問題が起きない。
  • 画像データからテクスチャの作成を行う関数 MakeTextureFromImage 
    int MakeTextureFromImage(TextureImage *tex, ImageData *img);
    この関数は img で指定された画像データから、テクスチャ tex を作成する。この関数は初期化を行う init 関数などで最初に一度だけ呼べばよい。

  • 画像ファイルからテクスチャの作成を行う関数 MakeTextureFromFile 
    int MakeTextureFromFile(TextureImage *tex, const char *filename);
    ファイル名 filename で指定された画像ファイルから、テクスチャ tex を作成する。この関数は初期化を行う init 関数などで最初に一度だけ呼べばよい。

  • テクスチャを四角形に貼り付けて画像を表示する関数 DrawTexturedQuad 
    void DrawTexturedQuad(TextureImage *tex, int xi, int yi, int w, int h);
    四角形の左下の座標 (xi, yi), 横幅 w と縦幅 h を指定し、四角形にテクスチャ tex を貼り付けて画像を表示する。この関数は描画を行う display 関数の中で使う。

ゲームプログラミングには総合的な知識が必要になる。例えば、数学 (ベクトル・行列、解析、統計)、アルゴリズムとデータ構造、OS などを含む計算機の仕組み、ネットワーク、物理、CG などである。これらは授業でしっかり学んでほしい。プログラミング言語としては、実行速度が重要なゲームでは C 言語や C++ 言語が用いられる。最近では Unity という統合開発環境を用いて JavaScript や C# などのプログラミング言語も使われている。C++ については、このページの下の参考資料に少しだけ紹介している。JavaScript や C# については大学の講義では扱わないので、基本的に独習してほしい。この他にも、どんなゲームを作るかによって必要な知識は変わってくる。

当たり判定 (衝突判定)

例えばシューティングゲームで「弾が敵に当たった」、アクションゲームで「自キャラが障害物に当たった」など、何かと何かが当たったかどうかを判定する処理がゲームでは欠かせない。この判定処理のことを「当たり判定」(衝突判定)と呼ぶ。基本的には、判定したいものを簡単な形状に置き換えて、簡単な形状同士で計算する。例えば 2 次元なら、箱 (長方形) と箱、円と円、円と線分、円と点などで当たり判定を計算する。具体的な計算方法はネットで調べてみてほしい。

包括的にゲームプログラミングを学べる書籍はあまり多くないと思われる。また、書籍はプログラミングがある程度できる人を対象としたものが多く、初心者にはとっつきにくい。例えば次の本は、ゲーム会社の新人研修で学ぶ知識をまとめた良書だが、C++ 言語についてある程度理解していることが前提になっている。

構造体を学習してからのレポート課題で、扱いたいデータを構造体で表現し、それを操作するための関数を作る、ということを繰り返してきた。このように、データとその操作方法、というのをセットにすると、プログラムを設計・開発の見通しがよくなることが知られている。例えば、構造体のメンバ変数に値をいちいち代入したり参照したりするのではなく、データに対するまとまった一連の操作を記述できれば、ソースコードが理解しやすくなり、ひいてはより抽象的なレベルでプログラムを設計・開発できるようになる。このような考え方に基づいたプログラミングの方法を、オブジェクト指向プログラミングという。現在ソフトウェア開発で標準的に用いられており、大規模なプログラムの開発時に大変有効である。
C 言語を拡張して、オブジェクト指向プログラミングをサポートするようになったのが、C++ 言語である。C 言語から変わっているところは多々あるが、例えば、構造体にメンバ変数だけでなく、その構造体を扱うための関数、「メンバ関数」を記述できるようになった。構造体を拡張した「クラス」という概念が導入されている。次のプログラムをコンパイル・実行してみよ。

#include <cstdio> // C++ では読み込むヘッダファイルが少し変わる
#include <cstring>
 
// C++ では構造体に typedef を付けなくてよい
struct Hello {
    char name[200]; // メンバ変数
     
    // 以下の関数は、メンバ関数
     
    void setName(const char *str)
    {
        strcpy(name, str);
    }
 
    void hello()
    {
        printf("Hello, %s!\n", name);
    }
};
 
int main(int argc, char *argv[])
{
    Hello h;
 
    h.setName("Yamada");
    h.hello();
 
    return 0;
}
[実行例]
$ g++  hello.cpp  -o  hello  ←※コンパイラには g++ を指定する。ファイルの拡張子は cpp にする。

$ ./hello
Hello, Yamada!

$

#include <iostream>
 
using namespace std;    // 名前空間 (namespace) というものを設定している
 
int main(int argc, char *argv[])
{
    int i;
 
    cout << "Input integer: ";    // cout は標準出力
    cin >> i;         // cin は標準入力
 
    cout << "Your input: " << i << endl;  // endl で改行
 
    return 0;
}
[実行例]

Input integer: 3	← 3 を入力する

$

    int g_NumCircles = 0;                   /* 円の数 */

    /* 円を描画する関数 */
    /* 中心点の座標が (xi,yi), 半径が radius, 色が (r,g,b) で与えられるものとする */
    void drawCircle(int xi, int yi, int radius, unsigned char r, unsigned char g, unsigned char b)
    {
        const int num_division = 64;
        int i;
     
        glColor3ub(r,g,b);
     
        glBegin(GL_TRIANGLE_FAN);
        glVertex2i(xi,yi);
        for (i=0; i<=num_division; i++)
        {
            const float rad = i * (2*M_PI) / (float)num_division;
            const float x = radius * cosf(rad) + xi;
            const float y = radius * sinf(rad) + yi;
            glVertex2f(x,y);
        }
        glEnd();
    }

        int i;

        /* ウィンドウを消去 … glClearColor で指定した色で塗りつぶし */
        glClear(GL_COLOR_BUFFER_BIT);
     
        /* for 文で円の個数だけ drawCircle 関数を呼び出して円を描画。自分で実装 */
     
        glutSwapBuffers();  /* ここまで指定した描画命令をウィンドウに反映 */
    }

レポートに関して類似した質問が多くあった場合、以下にその質問と回答を掲載します。定期的にチェックしてください。
Q. Windows 上で OpenGL を使うにはどうしたらいいですか？
A. Windows 10 以降なら "Bash on Windows (Windows Subsystem for Linux)" が使えます。それ以前の Windows なら Cygwin を使ってください。Cygwin は Windows 10 でも使えます。
[Bash on Windows]
まずネットの記事などを参照し、Bash on Windows をインストールできたとします。コンソールを開いて
$ sudo apt-get install freeglut3 freeglut3-dev
とやれば freeglut がインストールできます。これでコンパイルまではできるはずですが、実行しようとすると 
$ ./a.out
Error: Can't open display
などといったエラーが出るはずです。そこで [この URL] を参考に Xming X Server for Windows というのをインストールしてください。後は実習室の環境とほぼ同様に開発できるはずです。
[Cygwin] ※こちらは古い情報なのでうまくいかなければ教員にメールしてください。
下記ページの「17. CygwinでGLUTプログラムのリンクができない(OpenGLパッケージの問題)」に書いてある、OpenGL パッケージをインストールしてください。
  • 森北出版株式会社刊 「Springs of C」サポートwebページ
手順は上のリンク先に書いてある通りですが、次の通りに行ってください。
  1. Cygwin の setup プログラムを実行
  2. Select Packages のウィンドウの左下にある Hide obsolete packages のチェックを外す
  3. _obsolete と書かれているパッケージツリーの中から、"opengl: OpenGL-related libraries" を探す
  4. "opengl: OpenGL-related libraries" のところに "Keep" と書かれているのをクリックし、"Install" に変更
  5. 「次へ」をクリックしインストール
ここまでできたら、コンパイル・リンクを次のように行ってください。
[実行例] 
$ cc -c -Wall -I/usr/include/opengl *.c	 ←コンパイル。-I オプションのあとにスペースを入れない

$ cc *.o -lglut32 -lglu32 -lopengl32  ←リンク

$ ./a.exe  ←実行…Cygwin なので実行ファイルに .exe がつく
また、コンパイラに cc を使ってコンパイル時にエラーが出る場合は、cc の代わりに gcc-3 を使ってください。cc は gcc-4 へのシンボリックリンクになっています。
Q. Mac で GLUT を使うにはどうしたらいいですか。
A. Mac だとライブラリを framework というパッケージでまとめてあり、そちらを使う方法があります。GLUT の framework がインストールされているとして、次のようにコンパイルしてみてください。 
$ cc -o ColoredCircle ColoredCircle.o circles.o -framework GLUT -framework OpenGL
あるいは、次のようにしてみてください。 
$ cc -o ColoredCircle ColoredCircle.o circles.o -I/usr/X11R6/include L/usr/X11R6/lib -lglut -lGLU -lGL -lXmu -lXext -lX11 -lXi -lm
いずれの場合でも、ヘッダファイルをインクルードする場合は #include <GL/glut.h> の行を #include <GLUT/glut.h> に書き換える必要があるはずです。より詳しくは、2 年次向けの授業「コンピュータグラフィックス基礎 (CG 基礎)」第 1 回のところに説明がありますので参照してください。
Q. GLUT に必要なヘッダファイルやライブラリファイルはコピーしたのに、ソースコードをコンパイルしようとすると、次のようなエラーが出ます。
$ cc -c circles.c
circles.c:4:21: 致命的エラー: GL/glut.h: そのようなファイルやディレクトリはありません
A. コンパイラがヘッダファイルを探しに行けていないのが問題だと思われます。-I. (ハイフン、大文字の i 、ピリオドをくっつけたもの) をオプションに加えてコンパイルしてみてください。-I はコンパイラがヘッダファイルを探索する場所を指定するためのオプションです。ピリオドはカレントディレクトリを表します。 
$ cc -c circles.c -I.

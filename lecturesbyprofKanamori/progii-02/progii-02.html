配列とポインタ

教科書「やさしいC」の「10.1 配列とポインタの関係」「10.2 引数と配列」を読んで、配列とポインタのしくみを確認しよう。

配列とポインタの関係

配列名は配列の先頭要素のアドレスを表す

配列の配列名は、配列の先頭要素のアドレスを表す。つまり例えば、int a[3] という配列があったとき、配列名 a は配列の先頭要素のアドレス &a[0] を持つ。

思い出してみると、これまで scanf 関数などに char 型の配列を渡すとき 
char str[64];
scanf("%s", str);
などと書いていたが、これは実は、配列 str[ ] の先頭アドレス str (つまり &str[0]) を渡していた、ということである。64 バイトだけ確保されたメモリの先頭アドレスを scanf 関数に渡しており、scanf 関数はその先頭アドレスのところから値を書き込む。
前回の内容を復習すると、引数としてアドレスを受け取る関数に、ポインタを渡せることを学んだ。ポインタはアドレスを保持しているからである。では次のように書くとどうなるだろうか (前回の基本問題 1-4d)。 
char *p;	/* 初期化されていない…デタラメなアドレスが入っている */
scanf("%s", p);
これはセグメンテーション違反になる。なぜなら、ポインタ p が初期化されておらず、メモリの確保されていないデタラメなアドレスを指しており、scanf 関数がそのアドレスのメモリに書き込もうとしても書き込めないためである。ちゃんとポインタにメモリの確保された部分の先頭アドレスを代入し 
char str[64];
char *p = str;	/* ポインタにメモリの確保された部分の先頭アドレスを代入 */
scanf("%s", p);
などと書けば大丈夫である。

配列名とポインタの関係

実は、配列名とポインタはほぼ等価に扱える。配列の添字演算子 [ ] とポインタの間接参照演算子 * は、次のようにして互いに書き換え可能である。

例えば、配列 a[ ] の 2 番目の要素がほしいなら、a[2] と書いても *(a+2) と書いてもよい (ちなみに *(a+2) の a と 2 を入れ替えてもよいので、2[a] と書くこともできる…が混乱を招くので書かないこと)。ポインタ p に対して、添字演算子 [ ] を使って p[2] などと書いてもよい。
ただし、配列名とポインタには重要な違いがある。ポインタは保持しているアドレスを書き換えられるのに対し、配列名は保持しているアドレス (つまり配列の先頭アドレス) を書き換えられない。これは C 言語の仕様で、コンパイル時に配列がメモリ上のどこに配置されるかが決まり、その先頭アドレスを勝手に書き換えられないようにしてあるためである。要するに、配列名は定数アドレスで書き換え不可であり、ポインタが保持するアドレスは書き換え可、ということである。配列名とポインタの関係を次の表にまとめる。
項目配列名ポインタ
アドレスの代入不可可 (例: p = &a;)
添字演算子 [ ] の適用可可
間接参照演算子 * の適用可可
演算子 +, - の適用可 (例: *(a+1))可 (例: *(p+1))
演算子 ++, -- の適用
(アドレスを要素 1 つ分増減)		不可
(アドレスを書き換える操作のため)		可 (例: ++p, p--)
配列名は定数アドレスということで、例えば、配列をコピーしようとして次のようなコードを書いたとしてもコンパイルできない。 
int a[3] = { 1, 2, 3 };
int b[3];
b = a;	/* 配列 a の中身を配列 b にコピーしているつもり…しかしコンパイルエラー */
"b = a;" は、配列の中身のコピーではなく、「配列 b の先頭アドレスを配列 a の先頭アドレスで上書き」と解釈される。配列 b の先頭アドレスは書き換え不可のため、コンパイルエラーとなる。配列の中身をコピーするなら、for 文を使って要素をひとつずつコピーするか、strcpy 関数 (文字列専用) や memcpy 関数などを使う。
配列とポインタには、次のような密接な関係がある。
  • 配列 (例えば a[]) の配列名 (この場合は a) だけを記述すると、配列の先頭要素のアドレス (つまり &a[0]) を表せる
  • 配列名に * (間接参照演算子) をつけると、ポインタと同様に、先頭要素の値を参照できる
  • ポインタが配列のアドレスを指しているとき、添字演算子 [] で配列の要素を参照できる (ポインタが指しているアドレスが 0 番目になる)
  • 配列名には、ポインタと同様な、アドレスに関する演算が適用できる
  • ただし、配列名にはアドレスを代入することはできない (配列名が指すアドレスは書き換えられない)
ここで、ポインタに対して使える「アドレスに関する演算」とは、次の表のようなものである。
演算子例 (p, p1, p2 はポインタ)例の意味
+p1+1p が指している要素の次の要素のアドレスを得る
-p1-1p が指している要素の前の要素のアドレスを得る
p1-p2p1 と p2 の間の要素数を得る
++p++ (または ++p)p が指している要素の次の要素のアドレスを得る (p の値が要素 1 つ分増える)
--p-- (または --p)p が指している要素の前の要素のアドレスを得る (p の値が要素 1 つ分減る)
配列名に対しては、+ や - などの演算子で、アドレスを計算できる。ただし、++ や -- は、ポインタの持つアドレスを書き換える操作であり、配列名の持つアドレスは書き換え不可なので、配列名に対しては ++ や -- は使えない。
